   A trainer that implements SAC with a controller in the policy network.

   The controller is used to compute actions but without differentiating through it (SAC-ZOP).
   Uses parameter noise and a parameter critic.

   :ivar train_env: The training environment.
   :ivar q: The Q-function approximator (critic).
   :ivar q_target: The target Q-function approximator.
   :ivar q_optim: The optimizer for the Q-function.
   :ivar pi: The policy network containing the parameterized controller (the actor).
   :ivar pi_optim: The optimizer for the policy network.
   :ivar log_alpha: The log of the temperature parameter.
   :ivar alpha_optim: The optimizer for the temperature parameter.
                      If `None`, the temperature is fixed.
   :ivar target_entropy: The target entropy for the policy.
                         If `None`, the temperature is fixed.
   :ivar entropy_norm: The normalization factor for the entropy term.
                       Normalizes the entropy based on the ratio of parameter and action dimensions.
   :ivar buffer: The replay buffer used to store transitions.



   .. py:method:: act(obs, deterministic: bool = False, state: leap_c.controller.CtxType | None = None) -> tuple[numpy.ndarray, leap_c.controller.CtxType, dict[str, float]]

      Act based on the observation.

      This is intended for rollouts (= interaction with the environment).

      :param obs: The observation for which the action should be determined.
      :type obs: Any
      :param deterministic: If `True`, the action is drawn deterministically.
      :type deterministic: bool
      :param state: The state of the policy. Useful, if, e.g., the policy is recurrent or includes
                    an MPC planner which might want to pass warmstarting information.
                    Note, that at the start of an episode, the state is assumed to be `None`.

      :returns: The action, the state of the policy and potential solving stats.



   .. py:method:: periodic_ckpt_modules() -> list[str]

      Returns the modules that should be checkpointed periodically.

      This is used for example for tracking policy parameters over time.



   .. py:method:: singleton_ckpt_modules() -> list[str]

      Returns the modules that should be checkpointed only once.

      Replay Buffers often should not be stored multiple times as there is overlap.



   .. py:method:: train_loop() -> Generator[tuple[int, float], None, None]

      The main training loop.

      For simplicity, we use an Iterator here, to make the training loop as simple as
      possible. To make your own code compatible use the yield statement to return the
      number of steps your train loop did. If yield does not always return `1`, the validation
      might be performed not exactly at the specified interval.

      :Yields: The number of steps the training loop did.
               The training score over these steps.

.. py:class:: SacZopTrainerConfig

   Bases: :py:obj:`leap_c.trainer.TrainerConfig`


   Specific settings for the Zop trainer.

   :ivar critic_mlp: The configuration for the Q-networks (critics).
   :ivar batch_size: The batch size for training.
   :ivar buffer_size: The size of the replay buffer.
   :ivar gamma: The discount factor.
   :ivar tau: The soft update factor for the target networks.
   :ivar soft_update_freq: The frequency of soft updates (in steps).
   :ivar lr_q: The learning rate for the Q-networks.
   :ivar lr_pi: The learning rate for the policy network.
   :ivar lr_alpha: The learning rate for the temperature parameter.
                   Can be set to None to avoid updating the temperature.
   :ivar init_alpha: The initial temperature parameter.
   :ivar target_entropy: The minimum target entropy for the policy.
                         If `None`, it is set automatically depending on dimensions of the action space.
   :ivar entropy_reward_bonus: Whether to add an entropy bonus to the reward.
   :ivar num_critics: The number of critic networks.
   :ivar update_freq: The frequency of updating the networks (in steps).
   :ivar actor: Configuration for the HierachicalMPCActor.
