.. py:function:: episode_rollout(policy: Callable[[numpy.ndarray], tuple[numpy.ndarray, leap_c.controller.CtxType, dict[str, float] | None]], env: gymnasium.Env, episodes: int = 1, render_episodes: int = 0, render_human: bool = False, video_folder: str | pathlib.Path | None = None, name_prefix: str | None = None, rng: leap_c.torch.utils.seed.RngType | None = 1042, step_callback: Callable[[int, numpy.ndarray, numpy.ndarray, float, dict, Any], None] | None = None) -> Generator[tuple[dict[str, float | bool | list], dict[str, list]], None, None]

   Rollout episodes with the given policy.

   :param policy: The policy to be used for the rollout.
   :type policy: Callable
   :param env: The gym environment.
   :type env: Env
   :param episodes: The number of episodes to run.
   :type episodes: int
   :param render_episodes: Number of episodes to render. If `0`, no episodes will be rendered.
   :type render_episodes: int
   :param render_human: If `True`, render the environment in human mode.
                        The environment render mode should then also be human render mode.
                        Can not be `True` if `video_folder` is provided.
   :type render_human: bool
   :param video_folder: The environment is rendered and saved as a
                        video in this folder. Can not be set if `render_human` is `True`.
   :type video_folder: str | Path, optional
   :param name_prefix: The prefix for the video file names. Must be provided if
                       `video_folder` is provided.
   :type name_prefix: str, optional
   :param rng: The random number generator or seed for seeding the environment.
   :type rng: RngType, optional
   :param step_callback: Optional callable invoked after each env step with
                         ``(step, obs, action, reward, info)`` where ``step`` is the 1-based index
                         within the episode. Resets to 1 at the start of each episode.

   :Yields: The first dictionary containing the information about the rollout, at least containing the
            following keys:

            - ``"score"``: The cumulative reward of the episode,
            - ``"length"``: The length of the episode,
            - ``"terminated"``: Whether it terminated,
            - ``"truncated"``: Whether it truncated,
            - ``"inference_time"``: The average inference time of the policy per step.

            The second dictionary containing statistics returned by the policy.
