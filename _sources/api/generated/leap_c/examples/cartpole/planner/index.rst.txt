.. py:class:: CartPolePlanner(cfg: CartPolePlannerConfig | None = None, export_directory: pathlib.Path | None = None)

   Bases: :py:obj:`leap_c.planner.ParameterizedPlanner`\ [\ :py:obj:`leap_c.ocp.acados.torch.AcadosDiffMpcCtx`\ ]


   Acados-based planner for `CartPole`, aka inverted pendulum.

   The state and action correspond to the observation and action of the CartPole environment.
   The cost function takes the form of a weighted least-squares cost on the full state and action,
   and the dynamics correspond to the simulated ODE of the standard CartPole environment
   (using RK4). The inequality constraints are box constraints on the action and
   on the cart position.

   :ivar cfg: A configuration object containing high-level settings for the MPC problem,
              such as horizon length.

.. py:class:: CartPolePlannerConfig

   Configuration for the CartPole planner.

   :ivar N_horizon: The number of steps in the MPC horizon.
                    The MPC will have N+1 nodes (the nodes 0...N-1 and the terminal
                    node N).
   :ivar T_horizon: The simulation time between two MPC nodes will equal
                    T_horizon/N_horizon [s] simulation time.
   :ivar Fmax: Bounds of the box constraints on the maximum force that can be
               applied to the cart [N] (hard constraint)
   :ivar x_threshold: Bounds of the box constraints of the maximum absolute position
                      of the cart [m] (soft/slacked constraint)
   :ivar cost_type: The type of cost to use, either "EXTERNAL" or "NONLINEAR_LS".
   :ivar discount_factor: discount factor along the MPC horizon.
                          If `None`, it defaults to the behavior of `AcadosOcpOptions.cost_scaling`.
   :ivar n_batch_init: Initially supported batch size of the batch OCP solver.
                       Using larger batches will trigger a delay for the creation of more solvers.
                       If `None`, a default value is used.
   :ivar num_threads_batch_solver: Number of parallel threads to use for the batch OCP solver.
                                   If `None`, a default value is used.
   :ivar dtype: Type the planner output tensors will automatically be cast to. If `None`, PyTorch
                default dtype is used.
